01 · The audit

What reads premium, what looks unfinished

An honest pass over the current build. The bones are good — the discipline is already there. The gap is tactility: the product's signature object, the playing card, still reads as a placeholder rectangle.

Already reads premium

Keep and protect these.

  • ✓Palette discipline. Charcoal, ivory and a single muted gold — it never shouts. Rare for crypto.
  • ✓Editorial serif display against a clean UI sans — a confident, grown-up typographic voice.
  • ✓The trust story. Provably-fair hands and double-entry custody are real differentiators.
  • ✓Spacing rhythm. Generous, calm layout — the quiet-luxury intent is legible.

Looks unfinished

Where production value leaks out.

  • →Flat playing cards. The brand's signature object reads as a grey-on-ivory placeholder rectangle.
  • →An all-type hero. No object, no focal point — nothing to make arrival feel like an event.
  • →Felt with no depth. A single radial gradient — seats float on it; nothing feels tactile.
  • →Gold as flat fill. The solid-gold button reads as yellow paint, not restrained metal.
  • →One flat plane. Uniform borders and blur — no elevation hierarchy between surfaces.
  • →Glyph furniture. The dealer button is a letter; chips and pot are text — not objects on a table.

09 · Performance & fallback

Built to never cost a frame

The deliberate call: layered CSS 3D transforms, not a WebGL context per card. The card can appear dozens of times without a renderer, animates only transform/opacity on the compositor, and degrades cleanly. (react-three-fiber stays a future option for one bespoke hero scene — but it's the wrong default for a card that repeats.)

~3 KBgzipped, lazy-split

Bundle delta. The Card3D chunk is code-split out of the initial load. No three.js (~150 KB) — the WebGL path is avoided entirely.

60 fpsmid-tier laptop

Frame timings. Tilt, float and flip animate transform only — GPU-composited, zero layout or paint. will-change:transform on the card.

2 / 18live 3D at 9-max

9-max table. Board and seats stay 2D; only the winner's two cards go 3D at showdown. Hero and lobby moments are isolated screens.

graceful

Fallbacks. Reduced-motion drops tilt/float (card stays still). Coarse pointers skip the tilt listener. No 3D-transform support → the existing flat PlayingCard.

useCard3DCapable.ts — the gate
// Presentation-only. Never gates gameplay.
export function useCard3DCapable(onScreen = 1) {
  const reduce = useMediaQuery("(prefers-reduced-motion: reduce)");
  const fine   = useMediaQuery("(pointer: fine)");
  return useMemo(() => (
    CSS.supports("transform-style", "preserve-3d") &&
    !reduce &&
    (navigator.hardwareConcurrency ?? 4) >= 4 &&
    !navigator.connection?.saveData &&
    onScreen <= 4            // else fall back to 2D grid
  ), [reduce, fine, onScreen]);
}

A design spec for handoff — every surface here maps to an existing file: playing-card.tsx, poker-table-view.tsx, verify-hand-drawer.tsx, page.tsx. Engine and business logic untouched.
